# ADR-002: ログアウト時に GlobalSignOut による全トークン即時無効化を行わない

- **ステータス**: 決定済み
- **日付**: 2026-04-03
- **関連Issue**: #288

## コンテキスト

ログアウト時のトークン無効化方式として、以下の2つの選択肢を検討した。

| 方式                        | Cognito API     | 無効化対象                                   | 他デバイスへの影響         |
| --------------------------- | --------------- | -------------------------------------------- | -------------------------- |
| A. Refresh Token のみ無効化 | `RevokeToken`   | Refresh Token のみ                           | なし                       |
| B. 全トークン即時無効化     | `GlobalSignOut` | Access Token・ID Token・Refresh Token すべて | 全デバイスで強制ログアウト |

## 決定

**`GlobalSignOut` による全トークン即時無効化は行わない。**
ログアウト時は `RevokeToken` を使い、Refresh Token のみを無効化する。
Access Token は最大 60 分の自然失効を許容する。

## 理由

- 本サービスは個人利用を想定しており、`GlobalSignOut` による他デバイスの強制ログアウトはユーザー体験を損なう
- Access Token の TTL は 60 分と短く、Refresh Token を無効化することで新規トークン発行を防げるため、セキュリティ上の許容範囲内と判断した

## トレードオフ

- **デメリット**: ログアウト後最大 60 分間、流出した Access Token を使って API を叩かれるリスクが残る
- **代替対策**: Refresh Token を即時無効化することで、トークンの継続的な延命を防ぐ

## 再検討条件

複数デバイスでの利用が一般的になった場合、またはセキュリティ要件が強化された場合に再検討する。
