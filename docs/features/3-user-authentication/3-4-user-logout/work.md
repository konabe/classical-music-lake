# ユーザーログアウト

## 概要

フロントエンド側でユーザーのログアウト機能を実装し、認証情報を削除してログイン画面にリダイレクトする。

## スコープ

### フロントエンド

- ログアウトボタン（ナビゲーションバーなど）
- ログアウト処理（トークン削除）
- ログイン画面へのリダイレクト

### バックエンド

- なし（ログアウトはクライアント側の処理）

## 前提条件

- ユーザーログイン機能が完了（タスク 3-3）
- トークン保存場所が決定（localStorage/Cookie）

## テストケース

### フロントエンド

1. **ログアウトボタンが表示される**
   - ログイン状態時にのみ表示される
   - 適切なUIに配置されている

2. **ログアウト処理**
   - ログアウトボタン押下時、localStorage/Cookie 内のトークンが削除される

3. **リダイレクト**
   - ログアウト後、ログイン画面にリダイレクトされる

4. **ログイン画面での動作確認**
   - ログイン画面が表示される（再度ログイン可能）
   - 保護されたページ（視聴ログ一覧など）へのアクセスが制限される

## 依存関係

- `3-3-user-login` (トークン管理が必要)
