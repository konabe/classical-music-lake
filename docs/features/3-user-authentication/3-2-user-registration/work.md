# ユーザー登録

## 概要

フロントエンドのユーザー登録フォーム（Nuxt）とバックエンド Lambda 関数を実装し、ユーザーが新規登録できる機能を提供する。

## スコープ

### フロントエンド

- 登録フォーム（メールアドレス、パスワード）
- バリデーション（メール形式、パスワード強度）
- API 呼び出し（登録エンドポイント）
- エラーメッセージ表示
- 登録成功後のメッセージ（確認メール送信済み）

### バックエンド

- `POST /auth/register` エンドポイント
- リクエスト検証
- Cognito へのユーザー登録（`AdminCreateUser` または `SignUp`）
- エラーハンドリング（メール重複、パスワード不適切など）
- 正常系レスポンス

## 前提条件

- AWS Cognito User Pool が セットアップ済み（タスク 3-1）
- メール送信設定が有効

## テストケース

### フロントエンド

1. **フォームが表示される**
   - メールアドレス入力欄
   - パスワード入力欄
   - 登録ボタン

2. **入力値の検証**
   - 無効なメール形式でエラー表示
   - パスワード不足でエラー表示

3. **API 呼び出し**
   - 有効な値で登録ボタン押下時、API 呼び出し
   - ローディング状態表示

4. **成功時の挙動**
   - 成功メッセージ表示（「確認メールを送信しました」）
   - フォームリセット

5. **エラー時の挙動**
   - メール重複時にエラーメッセージ表示
   - その他のエラー（API エラー）表示

### バックエンド

1. **リクエスト検証**
   - メールアドレス必須、パスワード必須
   - メール形式チェック
   - パスワード長・複雑性チェック

2. **Cognito への登録**
   - ユーザーが Cognito に登録される
   - メール確認フローが開始される

3. **エラーハンドリング**
   - メール重複時: `UserExists` エラーで適切に応答
   - その他の Cognito エラーで 400/500 応答

4. **レスポンス**
   - 成功時: 201 Created、`{ message: "User created successfully" }`
   - エラー時: 適切なステータスコードとエラーメッセージ
