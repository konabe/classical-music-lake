# メールアドレス確認 - 要件定義

## 概要

ユーザー登録時にバックエンドから送信される認証コードをフロントエンドで入力・検証し、メールアドレスの確認を完了させる機能を実装する。

## 目的

- ユーザーが実在するメールアドレスを所有していることを確認する
- 確認済みユーザーのみがサービスを利用できるようにする

## 機能スコープ

### 実装する機能

- 認証コード入力画面
  - メールアドレス確認のための6桁（または任意桁）の認証コード入力フォーム
  - コードが正しい場合：自動的にログイン状態にして画面遷移する
  - コードが無効・期限切れの場合：エラーメッセージを表示する
  - 再送信ボタンを設け、認証コードを再発行できる

- ユーザー登録後の自動遷移
  - 登録完了直後に認証コード入力画面へ自動遷移する

- 未確認ユーザーのログイン制御
  - メールアドレス未確認のままログインしようとした場合、認証コード入力画面へ誘導する

### 実装しない機能（将来の実装対象）

- 認証コードの有効期限の延長・カスタマイズ
- SMS・電話による認証
- 複数メールアドレスの管理

## 設計申し送りメモ

- 現在のバックエンド（Cognito 等）が認証コード送信をどのエンドポイントで提供しているか確認すること
- 再送信 API のエンドポイントおよびレート制限の有無を確認すること
- 認証コード確認後に発行されるトークン（JWT 等）をフロントエンドでどう保持するか検討すること（cookie vs localStorage）
- 未確認状態の判定をフロントエンド・バックエンドのどちらで行うか設計で決定すること
