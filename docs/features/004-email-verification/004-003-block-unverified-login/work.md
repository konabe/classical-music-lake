# 004-003 未確認ユーザーのログイン制御

## 概要

メールアドレス未確認のユーザーがログインしようとした場合に、認証コード入力画面へ誘導する。

## ユーザーストーリー

> ユーザーとして、メール確認を完了していない状態でログインしようとしたとき、確認コード入力画面へ案内されることで、迷わずメール確認を完了できるようにしたい。

## 機能詳細

- ログイン試行時にバックエンドからメール未確認エラーが返された場合、認証コード入力画面へリダイレクトする
- リダイレクト時に入力したメールアドレスを認証コード入力画面へ引き渡す

## 受入テストケース

- [ ] メール未確認のユーザーがログインを試みると、認証コード入力画面へ遷移する
- [ ] 認証コード入力画面に、ログイン時に入力したメールアドレスが表示される
- [ ] メール確認済みのユーザーは通常通りログインでき、認証コード入力画面へは遷移しない
