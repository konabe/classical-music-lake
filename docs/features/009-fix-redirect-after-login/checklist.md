# 009-fix-redirect-after-login チェックリスト

## 概要

Issue #253「ログイン後に戻るボタンで無限ループが発生する」のバグ修正。

未認証ユーザーが保護ページにアクセスした際、`navigateTo("/auth/login")` でリダイレクトすると
ブラウザの履歴にリダイレクト元のURLが残るため、ログイン後に戻るボタンを押すと
保護ページ → ログインページ → 保護ページ... という無限ループが発生していた。

`replace: true` を指定してヒストリーを置き換えることで、戻るボタンによる無限ループを防ぐ。

## 受け入れ条件

- [x] 未認証ユーザーが保護ページにアクセスするとログインページにリダイレクトされる
- [x] ログイン後に戻るボタンを押しても無限ループが発生しない（`replace: true` でリダイレクト）
- [x] トークンがある場合はリダイレクトしない

## 変更ファイル

| ファイル                      | 変更内容                                   |
| ----------------------------- | ------------------------------------------ |
| `app/middleware/auth.ts`      | `navigateTo` に `replace: true` を追加     |
| `app/middleware/auth.test.ts` | auth middleware のユニットテストを新規追加 |
