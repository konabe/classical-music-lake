# 楽曲一覧のYouTubeサムネイル表示 - 要件定義

## 概要

楽曲マスタの一覧ページ（`/pieces`）で、楽曲に設定されている `videoUrl` が YouTube 動画の URL の場合、各楽曲行に YouTube 動画のサムネイル画像を表示する。サムネイルをクリックすると、その楽曲の詳細ページ（`/pieces/[id]`）へ遷移する。

## 目的

- 楽曲一覧を視覚的に認識しやすくし、動画が登録されている楽曲を一目で把握できるようにする
- サムネイルから直接詳細ページへ遷移できるようにすることで、動画試聴までの導線を短縮する
- クラシック音楽鑑賞において、曲の雰囲気を視覚的にも感じ取れる体験を提供する

## 機能スコープ

### 実装する機能

- 楽曲一覧ページ（`/pieces`）の各楽曲行に YouTube サムネイル画像を表示する
  - 楽曲の `videoUrl` が YouTube URL（`youtube.com/watch?v=...` または `youtu.be/...`）である場合のみサムネイルを表示する
  - サムネイル画像は YouTube の公式サムネイル URL（`https://img.youtube.com/vi/{videoId}/...`）を利用する
- サムネイルは楽曲行の左側に配置する
- サムネイルをクリックすると、該当楽曲の詳細ページ（`/pieces/[id]`）へ遷移する
- `videoUrl` が未設定、または YouTube 以外の URL の場合は、サムネイル領域には何も表示しない（従来の表示を維持）

### 実装しない機能（将来の実装対象）

- YouTube 以外の動画サービス（Vimeo、ニコニコ動画等）のサムネイル表示
- YouTube 以外の動画 URL に対する汎用プレースホルダーアイコンの表示
- サムネイルのホバー時プレビュー再生
- サムネイルサイズのユーザーカスタマイズ
- 楽曲詳細ページ以外（コンサート記録のプログラム表示等）でのサムネイル表示

## 受入テストケース

### AC-1: YouTube URL が設定された楽曲のサムネイル表示

- **前提**: `videoUrl` に YouTube URL（例: `https://www.youtube.com/watch?v=abcdefg`）が設定された楽曲が存在する
- **操作**: 楽曲一覧ページ（`/pieces`）を開く
- **期待結果**: 該当楽曲行の左側に YouTube サムネイル画像が表示される

### AC-2: 短縮形 YouTube URL（youtu.be）のサムネイル表示

- **前提**: `videoUrl` に短縮形 YouTube URL（例: `https://youtu.be/abcdefg`）が設定された楽曲が存在する
- **操作**: 楽曲一覧ページを開く
- **期待結果**: 該当楽曲行の左側に YouTube サムネイル画像が表示される

### AC-3: videoUrl 未設定の楽曲ではサムネイルが表示されない

- **前提**: `videoUrl` が未設定の楽曲が存在する
- **操作**: 楽曲一覧ページを開く
- **期待結果**: 該当楽曲行にサムネイル画像が表示されず、従来通り曲名・作曲家・カテゴリバッジのみが表示される

### AC-4: YouTube 以外の動画 URL の楽曲ではサムネイルが表示されない

- **前提**: `videoUrl` に YouTube 以外の URL（例: `https://example.com/video.mp4`）が設定された楽曲が存在する
- **操作**: 楽曲一覧ページを開く
- **期待結果**: 該当楽曲行にサムネイル画像が表示されず、従来通り曲名・作曲家・カテゴリバッジのみが表示される

### AC-5: サムネイルクリックで詳細ページへ遷移

- **前提**: YouTube サムネイルが表示されている楽曲が存在する
- **操作**: 該当楽曲のサムネイルをクリックする
- **期待結果**: 該当楽曲の詳細ページ（`/pieces/[id]`）へ遷移する

### AC-6: サムネイル表示時のレイアウト崩れがない

- **前提**: サムネイルあり・なしの楽曲が混在している
- **操作**: 楽曲一覧ページを開く
- **期待結果**: どちらのケースでもレイアウトが崩れず、既存の編集・削除ボタン等の操作が可能である

## 設計申し送りメモ

- 既存の `app/utils/video.ts` に `isYouTubeUrl` と `extractYouTubeVideoId` が定義されているので、これらを再利用してサムネイル URL を構築すること
- YouTube サムネイルの URL パターンは `https://img.youtube.com/vi/{videoId}/{quality}.jpg` の形式（`mqdefault.jpg`, `hqdefault.jpg`, `maxresdefault.jpg` など）で取得可能
- 影響範囲は `app/components/molecules/PieceItem.vue` が中心となる想定。必要に応じて新規の Atom コンポーネント（例: `YouTubeThumbnail`）を作成することも検討する
- 一覧画面ではスマホ表示時のレイアウト崩れに注意する。現状 `PieceItem.vue` は 600px 以下で縦積みになる
- サムネイル画像の読み込み失敗時のフォールバック方針は設計フェーズで検討する
- サムネイル画像には適切な `alt` 属性を設定し、アクセシビリティを確保すること
