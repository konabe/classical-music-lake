# ユーザーログイン

## 概要

フロントエンドのログインフォーム（Nuxt）とバックエンド Lambda 関数を実装し、ユーザーがメール/パスワードで認証できる機能を提供する。

## スコープ

### フロントエンド

- ログインフォーム（メールアドレス、パスワード）
- API 呼び出し（ログインエンドポイント）
- 返却されたアクセストークンの保存（localStorage または Secure Cookie）
- ログイン成功後のナビゲーション（ホーム/視聴ログ一覧ページへ）
- エラーメッセージ表示

### バックエンド

- `POST /auth/login` エンドポイント
- リクエスト検証（メール、パスワード）
- Cognito へのユーザー認証（`AdminInitiateAuth` または `InitiateAuth`）
- JWT アクセストークンの返却
- エラーハンドリング（ユーザー未検証、パスワード間違いなど）

## 前提条件

- AWS Cognito User Pool が セットアップ済み（タスク 003-1）
- ユーザー登録機能が完了（タスク 003-2）

## テストケース

### フロントエンド

1. **ログインフォームが表示される**
   - メールアドレス入力欄
   - パスワード入力欄
   - ログインボタン

2. **入力値の検証**
   - 空の値でエラー表示
   - 無効なメール形式でエラー表示

3. **API 呼び出し**
   - 有効な値でログインボタン押下時、API 呼び出し
   - ローディング状態表示

4. **成功時の挙動**
   - API から返却されたトークンが保存される（localStorage/Cookie）
   - ホームページ/視聴ログ一覧へナビゲート

5. **エラー時の挙動**
   - メール/パスワードが間違いの場合: エラーメッセージ表示
   - ユーザーが未確認の場合: エラーメッセージ表示
   - その他の API エラー: エラーメッセージ表示

### バックエンド

1. **リクエスト検証**
   - メールアドレス必須、パスワード必須

2. **Cognito との認証**
   - ユーザー認証が成功し、JWT トークンが返却される

3. **トークンの返却**
   - アクセストークンがレスポンスに含まれる
   - トークン形式が有効な JWT であること

4. **エラーハンドリング**
   - ユーザーが存在しない場合: 401 Unauthorized
   - パスワードが間違いの場合: 401 Unauthorized
   - メール確認が未完了の場合: 403 Forbidden + 明確なエラーメッセージ
   - その他の Cognito エラー: 500 Internal Server Error

5. **レスポンス形式**
   - 成功時: 200 OK、`{ accessToken: "eyJ...", tokenType: "Bearer" }`
   - エラー時: 適切なステータスコードとエラーメッセージ
