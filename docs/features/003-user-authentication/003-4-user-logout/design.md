# ユーザーログアウト - 設計書

## 概要

フロントエンド側でトークンを削除し、ログイン画面にリダイレクトする機能を実装する。

## システムフロー

```text
ユーザーがログアウトボタンをクリック
    ↓
[localStorage から アクセストークン削除]
[Cookie から リフレッシュトークン削除（Set-Cookie: Max-Age=0）]
    ↓
[ログイン画面へナビゲート]
    ↓
ユーザーはログイン画面で再度ログイン可能
```

## フロントエンド設計

### ページ・コンポーネント

- **ナビゲーションバー**: ログアウトボタン（ログイン状態時のみ表示）
  - 配置: ページヘッダーまたはドロップダウンメニュー
  - テキスト: 「ログアウト」

- **Composable**: `composables/useAuth.ts`
  - `logout()` 関数
    - localStorage からトークンを削除
    - 認証状態を更新
    - ログイン画面へナビゲート

### UI フロー

1. ユーザーがログアウトボタン押下
2. トークン削除処理（同期的に実行）
3. ログイン画面へ自動ナビゲート
4. 保護されたページへのアクセスは制限（認証チェックで拒否）

### 認証状態管理

トークンの有無で認証状態を判定（以下いずれかの方式）:

- **方式1**: Composable (`useAuth().isAuthenticated`)
- **方式2**: Nuxt Plugin で Pinia ストアに状態を保持
- **方式3**: localStorage のトークンから直接判定

**採用**: **方式1** （シンプル、Composable で一元管理）

### ページルーティング保護

**ミドルウェア或いはナビゲーションガード**: `middleware/auth.ts`（Nuxt 3）

- 保護対象ページ: `/listening-logs/**` など
- 認証状態を確認、未認証の場合は `/auth/login` へリダイレクト

## バックエンド設計

### ログアウト処理

- バックエンド処理なし（クライアント側完全処理）
- 理由: JWT トークンはステートレス、サーバー側でのセッション管理不要

### Cognito との連携

- 実装不要（将来フェーズで Refresh Token の無効化が必要な場合は実装）

## セキュリティ考慮事項

### 実装予定

- localStorage からの完全削除
- 保護されたページへの自動リダイレクト
- Back ボタン対策: 保護されたページに `Cache-Control: no-store` を設定し、ログアウト後はキャッシュされたページの表示（バック遷移による履歴表示）を禁止する。再アクセス時は必ず再認証を要求する。

### 実装しない

- サーバー側セッション無効化（JWT ステートレス）
- Token Blacklist（将来フェーズ）

## トレードオフ・判断理由

| 項目                     | 選択                       | 理由                                                        |
| ------------------------ | -------------------------- | ----------------------------------------------------------- |
| **ログアウト処理の場所** | クライアント側のみ         | JWT ステートレス、サーバー実装不要                          |
| **トークン無効化**       | 実装しない（MVP）          | トークン有効期限 60 分で対応、将来 Token Blacklist 実装可能 |
| **UI/UX**                | ナビゲーションバーのボタン | 標準的、ユーザーが認識しやすい                              |
