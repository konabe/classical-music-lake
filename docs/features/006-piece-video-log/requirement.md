# 楽曲動画リンクと視聴中ログ記録 - 要件定義

## 概要

楽曲マスタ（Piece）に動画 URL を紐づけ、動画を視聴しながら気軽に鑑賞ログを記録できるようにする。

## 目的

- 楽曲を「探索する」体験を提供する（動画で曲を確認 → そのまま記録）
- 視聴ログ作成の摩擦を減らす（曲名・作曲家の手入力が不要になる）
- 動画視聴と記録を一体化し、鑑賞体験をシームレスに残せるようにする

## 機能スコープ

### 実装する機能

#### 1. Piece への動画 URL 追加

- 楽曲マスタに動画 URL フィールドを追加（任意項目）
- 楽曲マスタの新規作成・編集フォームで URL を入力できる
- バリデーションは URL フォーマットの確認のみ行う（ドメイン制限は行わない）
- YouTube 以外の URL も保存できる（将来的な他サービス対応を見越して制限しない）

#### 2. Piece 詳細ページ（`/pieces/[id]`）の新設

- YouTube URL が設定されている場合、動画プレイヤーを埋め込み表示する
- YouTube 以外の URL が設定されている場合、動画は埋め込まず外部リンクとして表示する（クリックで別タブ開）
- 動画再生を開始した時点で、メモ入力エリア（クイックログフォーム）が表示される
  - 外部リンクの場合はリンクをクリックした時点でクイックログフォームを表示する
- クイックログフォームは作曲家・曲名を Piece から自動入力済みの状態で表示される
- いつでも「記録する」で視聴ログとして保存できる（視聴日時は保存時点の日時を使用）

#### 3. 視聴ログ作成フォームでの動画表示

- 楽曲マスタから曲を選択した際、その曲に動画 URL が設定されていれば動画エリアを表示する
- YouTube URL の場合は埋め込みプレイヤーを表示する
- YouTube 以外の URL の場合は外部リンクとして表示する（クリックで別タブ開）

### 実装しない機能（将来の実装対象）

- YouTube 以外のサービス（Spotify 等）の埋め込みプレイヤー対応
- 動画の再生位置と記録のタイムスタンプの紐づけ
- 楽曲マスタ一覧ページでの動画有無フィルタリング

## 設計申し送りメモ

- 動画 URL フィールドの名前は `videoUrl` とする（`youtubeUrl` は避け、将来の拡張に備える）
- 動画の種別判定（例: YouTube か否か）はフロントエンドで URL パターンを見て行う
- YouTube の埋め込みには `/embed/{videoId}` 形式を使用する
- クイックログは通常の視聴ログ（`ListeningLog`）として保存する（専用エンティティは不要）
- Piece 詳細ページは既存の `PieceList` から各アイテムへのリンクを追加して導線を作る
