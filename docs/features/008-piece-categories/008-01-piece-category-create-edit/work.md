# ワーク1: 楽曲カテゴリの作成・編集対応

## ユーザーストーリー

楽曲マスタの新規作成・編集時に、ジャンル・時代・編成・地域の4軸のカテゴリを設定できるようにする。

## 要件

- Piece 型に以下の optional フィールドを追加する
  - `genre`（ジャンル）: 交響曲、協奏曲、室内楽、独奏曲、歌曲、オペラ、宗教音楽、その他
  - `era`（時代）: バロック、古典派、ロマン派、近現代、その他
  - `formation`（編成）: ピアノ独奏、弦楽四重奏、管弦楽、声楽、その他
  - `region`（地域）: ドイツ・オーストリア、フランス、ロシア、イタリア、その他
- 各カテゴリは固定の選択肢（Union type）から選ぶ
- 各カテゴリは任意入力（未設定も許容）
- 既存データとの後方互換性を保つ（optional フィールドのため既存データに影響なし）
- フロントエンドとバックエンドの両方の型定義を更新する
- バックエンドAPIで新フィールドの保存・更新に対応する
- フロントエンドの新規作成フォーム・編集フォームにカテゴリ選択UIを追加する

## 受入テストケース

- [ ] 楽曲新規作成時に、4軸のカテゴリをそれぞれ選択して保存できる
- [ ] 楽曲新規作成時に、カテゴリを未選択のまま保存できる（既存動作を壊さない）
- [ ] 楽曲新規作成時に、一部のカテゴリのみ選択して保存できる
- [ ] 楽曲編集時に、カテゴリを変更して保存できる
- [ ] 楽曲編集時に、設定済みのカテゴリを未選択に戻して保存できる
- [ ] 各カテゴリの選択肢が仕様通りの固定値であること
- [ ] カテゴリフィールドに不正な値を送信した場合、バリデーションエラーになること
- [ ] 既存の楽曲データ（カテゴリなし）が引き続き正常に取得・更新できること
