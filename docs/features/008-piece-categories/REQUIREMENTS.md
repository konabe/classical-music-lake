# 楽曲マスタへのカテゴリ追加 - 要件定義

## 概要

楽曲マスタ（Piece）に複数軸のカテゴリ情報を追加し、クラシック音楽の楽曲を多面的に分類できるようにする。

## 目的

- 楽曲を「どんな種類か」「いつの時代か」「どんな編成か」「どの地域か」で分類することで、視聴ログの振り返りや絞り込みを助ける
- 将来的な検索・フィルタリング機能の基盤を整える

## 機能スコープ

### 実装する機能

- 楽曲マスタに以下の4軸のカテゴリフィールドを追加する
  - **ジャンル（genre）**: 交響曲、協奏曲、室内楽、独奏曲、歌曲、オペラ、宗教音楽、その他
  - **時代（era）**: バロック、古典派、ロマン派、近現代、その他
  - **編成（formation）**: ピアノ独奏、弦楽四重奏、管弦楽、声楽、その他
  - **地域（region）**: ドイツ・オーストリア、フランス、ロシア、イタリア、その他
- 各カテゴリは固定の選択肢（Enum）から選ぶ
- 各カテゴリは任意入力（未設定も許容）
- 楽曲の新規作成・編集フォームでカテゴリを設定できる
- 楽曲一覧・詳細でカテゴリ情報を表示する

### 実装しない機能（将来の実装対象）

- カテゴリによる検索・フィルタリング
- カテゴリの選択肢のカスタマイズ・追加
- 視聴ログへのカテゴリ付与

## 設計申し送りメモ

- 各カテゴリは Piece 型に optional フィールドとして追加する（既存データの後方互換性を保つ）
- 選択肢は型定義（Union type）として管理する
- DynamoDB のスキーマ変更は不要（新フィールドを追加するのみ）
- フロントエンドの `types/index.ts` とバックエンドの `backend/src/types/index.ts` の両方を更新すること
