# ワーク015-03: 管理者向け UI のロール判定ガード

## ユーザーストーリー

- 一般ユーザーとして、自分が触れない管理系の導線は見えないほうが画面がスッキリして使いやすい。
- 管理者として、ログインしている間は管理者向けの導線・操作 UI がこれまでどおり利用でき、楽曲マスタの管理ができる。

## 要件

### 機能要件

- ログイン中ユーザーが管理者ロールを持っているかどうかを判定する手段をフロントエンドに備える
- TOP ページの「管理者向けリンクセクション」は、管理者ロール保有者のみに表示する
  - 一般ユーザー・未ログインユーザーからはセクションごと見えない
- 楽曲マスタ関連画面の管理者操作 UI は、管理者ロール保有者のみに表示する
  - 対象: 楽曲マスタの「新規作成」ボタン、各楽曲の「編集」「削除」ボタン等の書き込み導線
  - 楽曲マスタの一覧・詳細の閲覧は従来どおり誰でも可能
- 管理者専用ページ（楽曲マスタの新規作成・編集ページ等）に非管理者が直接 URL でアクセスした場合、管理者向け UI を描画せず適切に遷移（例: TOP へリダイレクト）させる
- 管理者としてログイン→ログアウトしたあと、ページを再読み込みしなくても管理者向けセクション・UI が非表示になる（認証状態の変化に追従する）

### 非機能要件

- ロール判定はサーバー発行のトークンのクレームに基づく（クライアント側の固定値やメールアドレスベースの判定にしない）
- UI でガードされていても API 側（ワーク015-02）が 403 を返す前提で設計する（クライアント側ガードはあくまで UX 向上のためで、セキュリティ境界はサーバー側）
- `ARCHITECTURE.md` の該当箇所を更新し、管理者ロールによる UI 差分について記載する
- ワーク015-01 で admin グループが存在している前提で動作する

### スコープ外

- 管理者である旨を示すバッジ・アイコンの追加（必要になったら別ワーク）
- 管理者による他ユーザー管理 UI
- 視聴ログ・コンサート記録など個人データ画面の UI 変更

## 受入テストケース

### 管理者ユーザー

- [ ] 管理者ユーザーでログインすると、TOP ページの「管理者向けリンクセクション」が表示される
- [ ] 管理者ユーザーでログインすると、楽曲マスタ一覧画面に「新規作成」ボタンが表示される
- [ ] 管理者ユーザーでログインすると、楽曲マスタ詳細画面に「編集」「削除」ボタンが表示される
- [ ] 管理者ユーザーが楽曲マスタの新規作成・編集・削除を UI から実行できる
- [ ] 管理者ユーザーが楽曲マスタの新規作成／編集ページに直接 URL でアクセスでき、フォームが表示される

### 一般ユーザー

- [ ] 一般ユーザー（admin に属さないログインユーザー）で TOP ページを開くと「管理者向けリンクセクション」が表示されない
- [ ] 一般ユーザーで楽曲マスタ一覧画面を開くと「新規作成」ボタンが表示されない
- [ ] 一般ユーザーで楽曲マスタ詳細画面を開くと「編集」「削除」ボタンが表示されない
- [ ] 一般ユーザーが楽曲マスタの新規作成・編集ページに直接 URL でアクセスしても、管理者向け UI が描画されずに TOP などへ遷移する

### 未ログインユーザー

- [ ] 未ログインで TOP ページを開くと「管理者向けリンクセクション」が表示されない
- [ ] 未ログインで楽曲マスタ一覧・詳細を閲覧でき、書き込み系の UI は表示されない

### 状態遷移

- [ ] 管理者としてログイン中に表示されていた管理者向けセクション・ボタンが、ログアウト後（ページ再読み込みなしでも）非表示になる
- [ ] 一般ユーザーをログアウトしたあとに管理者として再ログインすると、管理者向けセクション・ボタンが表示される

### ドキュメント

- [ ] `ARCHITECTURE.md` に管理者ロールによる UI 差分が記載されている
