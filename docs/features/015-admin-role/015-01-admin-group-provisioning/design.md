# 設計: ワーク015-01 管理者グループのプロビジョニングと付与運用整備

## 概要

Cognito User Pool に `admin` グループを IaC（CDK）で追加し、運用者が AWS CLI/コンソール経由で任意ユーザーを管理者に昇格・降格できる基盤を整える。  
このワーク単体のリリースでアプリケーションの挙動は変わらない。

## CDK 設計

### Cognito グループの追加

`CfnUserPoolGroup` を使って `admin` グループを作成する。  
既存の `userPool` 定義の直後、同じスコープ内に追記する。

**設定方針:**

| 項目                 | 値                         | 理由                                            |
| -------------------- | -------------------------- | ----------------------------------------------- |
| グループ名           | `admin`                    | 要件で固定されている                            |
| 説明（description）  | 管理者グループを示す文字列 | 運用時に AWS コンソールで識別しやすくする       |
| precedence（優先度） | 設定しない（省略）         | 複数グループ間の優先順位が不要なため            |
| IAM ロール紐付け     | しない                     | Lambda 側でクレームを直接検証する設計のため不要 |

**環境分岐:**  
`userPool` 自体が `stageName` ごとに別リソースとして作成されるため、`CfnUserPoolGroup` を `userPool.userPoolId` に紐付けるだけで自動的に環境ごとに分離される。追加の分岐ロジックは不要。

**削除ポリシー:**  
`CfnUserPoolGroup` には個別の RemovalPolicy は設定しない。親リソースである `UserPool` の削除ポリシー（prod: `RETAIN` / dev,stg: `DESTROY`）に従う。

## ドキュメント設計

### `docs/OPERATIONS.md` への追記

既存の「Cognito ユーザー管理」セクションの下に「管理者ロール管理」セクションを追加する。

**記載内容:**

1. **管理者の付与（CLI）**  
   `aws cognito-idp admin-add-user-to-group` コマンドの使い方（User Pool ID、グループ名 `admin`、ユーザー名の指定方法）

2. **管理者の剥奪（CLI）**  
   `aws cognito-idp admin-remove-user-from-group` コマンドの使い方

3. **所属確認（CLI）**  
   `aws cognito-idp admin-list-groups-for-user` コマンドで、特定ユーザーが `admin` グループに属しているか確認する方法

4. **グループ全体の確認（CLI）**  
   `aws cognito-idp list-users-in-group` コマンドで `admin` グループのメンバー一覧を確認する方法

5. **トークンへの反映タイミング**  
   グループ変更は再ログイン後に ID Token の `cognito:groups` クレームへ反映されることを明記

6. **User Pool ID の取得方法**  
   CloudFormation スタック出力 `CognitoUserPoolId` から取得する手順を記載

### `docs/SPEC.md` への追記

「5.1 AWSリソース > Cognito」セクションに以下を追加する。

- `admin` グループが存在すること
- グループに所属するユーザーの ID Token には `cognito:groups: ["admin"]` クレームが付与されること
- グループの付与・剥奪は AWS CLI/コンソールによる手動運用であること

---

## 非機能・安全性

- **既存機能への影響なし**: Cognito グループの追加は既存 User Pool の設定を変更しない。既存ユーザーはいずれも `admin` グループに属さない初期状態になる
- **アプリケーション動作への影響なし**: グループ情報はトークンのクレームに追加されるだけで、現時点でアプリ側はそのクレームを参照しない
- **IaC 管理**: `CfnUserPoolGroup` を CDK で定義するため dev/stg/prod 全環境で再現可能

## 受入テストへの対応

| テストケース                                 | 対応方針                                                               |
| -------------------------------------------- | ---------------------------------------------------------------------- |
| 各環境に `admin` グループが存在する          | CDK デプロイ後に AWS コンソールで確認                                  |
| 既存ユーザーはグループに属さない             | Cognito の新規グループは初期メンバーゼロ                               |
| 運用手順で追加/削除/確認ができる             | OPERATIONS.md の手順に従い手動確認                                     |
| 再ログイン後にトークンにクレームが付与される | `admin-add-user-to-group` 後に再ログインし ID Token をデコードして確認 |
| 剥奪後はクレームが消える                     | `admin-remove-user-from-group` 後に再ログインして確認                  |
| 既存機能が従来どおり動作する                 | 既存テスト（ユニット・統合）の通過を確認                               |
