# ワーク015-01: 管理者グループのプロビジョニングと付与運用整備

## ユーザーストーリー

プロダクトオーナー（運用者）として、特定のユーザーを「管理者」として識別できるグループをサービスに用意し、AWS コンソールまたは CLI から任意のユーザーを管理者に昇格・降格できる。

## 要件

### 機能要件

- サービス上に「管理者ロール」を識別するためのグループ（`admin`）を用意する
- 運用者が AWS コンソールまたは CLI を使って、任意のユーザーを `admin` グループに追加／`admin` グループから削除できる
- `admin` グループに所属しているユーザーの認証トークンに、管理者であることを示すクレーム（`cognito:groups` に `admin` を含む）が付与される
- 運用手順が `docs/OPERATIONS.md` に記載されており、オペレータが手順を追うだけで管理者の付与・剥奪・確認ができる

### 非機能要件

- `admin` グループの作成は IaC（CDK）で管理し、環境（dev/stg/prod）ごとに再現可能にする
- このワークのみをリリースしてもアプリケーションの挙動は変わらない（管理者限定機能の実装はワーク015-02／015-03 で行う）
- 既存ユーザーのログイン・新規登録・視聴ログ／コンサート記録の利用に影響を与えないこと

### スコープ外

- 管理者限定 API の実装（ワーク015-02 で扱う）
- 管理者限定 UI の実装（ワーク015-03 で扱う）
- 画面からの管理者付与機能

## 受入テストケース

### グループのプロビジョニング

- [ ] dev／stg／prod それぞれの環境デプロイ後、Cognito User Pool に `admin` グループが存在する
- [ ] 既存ユーザーはいずれも `admin` グループに属していない（初期状態は管理者ゼロ）

### 管理者の付与・剥奪

- [ ] 運用ドキュメントの手順に従って、任意のユーザーを `admin` グループへ追加できる
- [ ] 運用ドキュメントの手順に従って、任意のユーザーを `admin` グループから削除できる
- [ ] 運用ドキュメントの手順に従って、特定ユーザーが `admin` グループに属しているかを確認できる

### クレームの反映

- [ ] `admin` グループに追加されたユーザーが再ログインすると、アプリで取得できる認証トークンのクレームに管理者であることを示す情報が含まれる
- [ ] `admin` グループから削除されたユーザーが再ログインすると、トークンから管理者の情報が取り除かれる

### 既存機能への影響

- [ ] 一般ユーザーの新規登録・メール確認・ログイン・ログアウトが従来どおり動作する
- [ ] 視聴ログ・コンサート記録・楽曲マスタ閲覧など既存機能が従来どおり動作する

### ドキュメント

- [ ] `docs/OPERATIONS.md` に「管理者ロールの付与・剥奪手順」が追記されている
- [ ] `docs/SPEC.md` に管理者グループの存在・役割が記載されている
