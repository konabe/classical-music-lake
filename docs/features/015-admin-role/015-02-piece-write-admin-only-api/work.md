# ワーク015-02: 楽曲マスタ書き込み API の管理者限定化

## ユーザーストーリー

サービス運営者として、楽曲マスタの書き込み操作（作成・更新・削除）を管理者のみが実行できるようにしたい。これにより、一般ユーザーや未認証の第三者によるマスタデータの意図しない変更を防ぎたい。

## 要件

### 機能要件

- 楽曲マスタの書き込み API を管理者ロール保有者のみ実行可能とする
  - 対象: `POST /pieces` / `PUT /pieces/{id}` / `DELETE /pieces/{id}`
- 管理者ロールを持たないユーザー、および未認証のリクエストに対しては `403 Forbidden` を返す
- エラーレスポンスのボディは既存のエラー形式に揃える（例: `{ "message": "..." }`）
- 楽曲マスタの参照 API（`GET /pieces` / `GET /pieces/{id}`）は従来どおり認証不要で公開のままとする
- 視聴ログ・コンサート記録など、既存の書き込み API の挙動は変更しない

### 非機能要件

- 認可チェックはサーバー側で確実に行う（フロントエンドのガードには依存しない）
- `SPEC.md` に以下を反映する
  - 楽曲マスタ書き込み API が管理者限定であること
  - 403 レスポンスの仕様
  - 管理者ロールと Cognito グループの関係
- ワーク015-01 のグループが存在している前提で動作する（グループが未作成でも API は落ちない）

### スコープ外

- フロントエンドの UI ガード（ワーク015-03 で扱う）
- 管理者による楽曲マスタ一覧／詳細 API への制限（参照は公開のまま）
- 書き込み API の操作ログ記録

## 受入テストケース

### 管理者としての書き込み

- [ ] `admin` グループに所属するユーザーの認証トークンを用いると `POST /pieces` で楽曲マスタを作成できる
- [ ] `admin` グループに所属するユーザーの認証トークンを用いると `PUT /pieces/{id}` で楽曲マスタを更新できる
- [ ] `admin` グループに所属するユーザーの認証トークンを用いると `DELETE /pieces/{id}` で楽曲マスタを削除できる

### 非管理者としての書き込み拒否

- [ ] 一般ユーザー（`admin` に属さないログインユーザー）の認証トークンで `POST /pieces` を呼ぶと `403 Forbidden` が返る
- [ ] 一般ユーザーの認証トークンで `PUT /pieces/{id}` を呼ぶと `403 Forbidden` が返る
- [ ] 一般ユーザーの認証トークンで `DELETE /pieces/{id}` を呼ぶと `403 Forbidden` が返る
- [ ] 認証ヘッダーなし（未認証）で `POST /pieces` を呼ぶと `403 Forbidden`（もしくは `401 Unauthorized`）が返り、データは変更されない
- [ ] 403 を受けた場合でも、楽曲マスタのデータが書き換わっていない

### 参照 API の無変更

- [ ] 未認証でも `GET /pieces` で楽曲マスタ一覧を取得できる
- [ ] 未認証でも `GET /pieces/{id}` で楽曲マスタ詳細を取得できる
- [ ] 一般ユーザーのトークン付きでも同様に `GET /pieces*` が取得できる

### 既存機能への非影響

- [ ] 視聴ログ API（作成・更新・削除）が従来どおり動作する
- [ ] コンサート記録 API（作成・更新・削除）が従来どおり動作する
- [ ] 認証系 API（登録・ログイン・メール確認・トークン更新）が従来どおり動作する
