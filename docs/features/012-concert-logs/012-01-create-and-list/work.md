# ワーク012-01: コンサート記録の作成・一覧

## ユーザーストーリー

ユーザーとして、コンサートの日時・会場・指揮者・オーケストラ・ソリストを入力して記録を作成し、一覧で確認できる。

## 要件

### 機能要件

- コンサート記録の新規作成ができる
- コンサート記録の一覧を表示できる
- 一覧はコンサート日時の降順（新しい順）で表示する
- ログインユーザーのみがアクセスできる（認証必須）
- ユーザーごとにデータが分離される

### 記録項目

| フィールド                  | 必須 | 説明                                 |
| --------------------------- | ---- | ------------------------------------ |
| 日時                        | 必須 | コンサートの開催日時                 |
| 会場                        | 必須 | コンサート会場名（テキスト自由入力） |
| 指揮者                      | 任意 | 指揮者名（テキスト自由入力）         |
| オーケストラ / アンサンブル | 任意 | 団体名（テキスト自由入力）           |
| ソリスト                    | 任意 | ソリスト名（テキスト自由入力）       |

> **注**: プログラム（演奏曲目）の楽曲マスタ連携はワーク012-03で実装する。本ワークでは含めない。

### 非機能要件

- 既存の視聴ログと同じ認証方式（Cognito Authorizer）を使用する
- DynamoDB テーブルは視聴ログとは別テーブルとする
- トップページからコンサート記録一覧への導線を設ける

## 受入テストケース

### 作成

- [ ] ログイン状態で、コンサート記録の新規作成画面にアクセスできる
- [ ] 日時・会場を入力して記録を作成できる
- [ ] 指揮者・オーケストラ・ソリストを任意で入力して記録を作成できる
- [ ] 日時が未入力の場合、バリデーションエラーになる
- [ ] 会場が未入力（空文字・空白のみ）の場合、バリデーションエラーになる
- [ ] 作成後、一覧画面に遷移する

### 一覧

- [ ] ログイン状態で、コンサート記録の一覧画面にアクセスできる
- [ ] 自分が作成したコンサート記録のみが表示される
- [ ] コンサート日時の降順で表示される
- [ ] 一覧に日時・会場が表示される

### 認証

- [ ] 未ログイン状態でコンサート記録の一覧・作成画面にアクセスすると、ログイン画面にリダイレクトされる

### ナビゲーション

- [ ] トップページからコンサート記録一覧への導線が存在する
