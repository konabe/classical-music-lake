# 設計について

## 1. 事前準備

- ブランチ名から現在の`{feature-title}`の内容を確認する

## 2. 一斉設計

- `work.md`をもとに、`design.md`を **`work.md` と同じディレクトリ**（`/docs/features/{index}-{feature-title}/{index}-{index-of-work}-{work}/design.md`）に作成する。
- `design.md`にレビュー結果の記載を求める。

## 3. レビュー結果の反映

- `design.md`に記載されたレビュー結果をもとに修正する。

## 4. mainブランチへPRを発行

- mainブランチに対して PRを発行する

## `design.md`の扱い

- 設計内容を記載する
- 最後にレビュー結果を記載する欄を作成する。
- 具体的なコードの記載はレビュー負荷が高いため禁止する。
