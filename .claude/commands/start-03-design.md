# 設計について

## 1. 事前準備

- 設計したいワークを尋ねる。

## 2. 設計

- `work.md`をもとに、`work.md` と同じディレクトリに `design.md` を作成する。
- `design.md`にレビュー結果の記載を求める。

## 3. レビュー結果の反映

- `design.md`に記載されたレビュー結果をもとに修正する。

## 4. mainブランチへPRを発行

- mainブランチに対して PRを発行する

## `design.md`の扱い

- 設計内容を記載する
- 最後にレビュー結果を記載する欄を作成する。
- 具体的なコードの記載はレビュー負荷が高いため禁止する。
